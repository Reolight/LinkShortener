Решение задачки на сокращатель ссылок

Следовал архитектуре onion.

Front — React, Backend Asp.Net Core 7
БД - MariaDB 11

Сервисы:
1. IUrlShortenerService - интерфейс сервиса для сокращения ссылок.
Текущая реализация считает MD5 полной ссылки и делает тем самым сокращённую ссылку не простой предсказуемой последовательностью
2. IUrlRepository — интерфейс сервиса для работы с бд через разные ORM. 
EfCoreRepository - работает с MariaDb через EFCore (по умолчанию).
NhibernateRepository - работает с NHibernate, однако настроить не смог: не может загрузить набор символов из бд (насколько я понял).
Тем не менее, для переключения на nhibernate можно использовать аргумент командной строки -nhibernate

EFCore не поддерживает автоматические миграции.
Поскольку NHibernate так и не заработал, эксперименты с его авто-миграциями не проводились.

Для миграции БД с EFCore в корневой папке решения через терминал необходимо выполнить: 
dotnet ef database update -s ./LinkShortener -p ./EFCoreStore 
-s - запускающийся проект
-p - проект с базой данных

N.B.: В ConnectionString (appsettings.json) прописан порт 3305.
